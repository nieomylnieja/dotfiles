# Sources

Research references used to compile the shell scripting skill.

## Style Guides

- [Google Shell Style Guide](https://google.github.io/styleguide/shellguide.html) —
  comprehensive shell coding standards covering naming,
  formatting, features, and when to use shell vs. other languages
<!-- markdownlint-disable-next-line MD013 -->
- [GNU Coding Standards — Command-Line Interfaces](https://www.gnu.org/prep/standards/standards.html#Command_002dLine-Interfaces) —
  conventions for `--help`, `--version`, exit codes,
  and option parsing

## Tools

- [shellcheck](https://www.shellcheck.net/) —
  static analysis tool for shell scripts;
  [wiki with all rules](https://www.shellcheck.net/wiki/)
- [shfmt](https://github.com/mvdan/sh) —
  shell script formatter supporting bash, POSIX sh, and mksh

## References

- [Bash Reference Manual](https://www.gnu.org/software/bash/manual/bash.html) —
  official GNU Bash documentation
<!-- markdownlint-disable-next-line MD013 -->
- [POSIX Shell Command Language](https://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html) —
  POSIX specification for the shell language
- [Bash Pitfalls](https://mywiki.wooledge.org/BashPitfalls) —
  comprehensive list of common bash scripting mistakes
- [Bash FAQ](https://mywiki.wooledge.org/BashFAQ) —
  solutions to frequently asked bash questions
